std::vector< BVHNode > build_bvh ( std::vector< CPU_Triangle > & tris)

Builds a simple median-split BVH from CPU triangles.

The resulting BVH uses a binary tree with splitting based on the longest axis of the node bounding box, partitioned by median position.

Parameters
trisInput/output triangle list. Order may be modified.
Returns
Linear array of BVHNode, representing the flattened tree.

High-level helper for loading a model and building its BVH.

Loads a model from disk, extracts triangles, builds a BVH, and uploads the resulting nodes and triangle data into GPU buffers/TBOs.

Old BVH data and the previous model (if any) are deleted and replaced.

Parameters
pathFile path to the model to load.
modelTransformTransform applied to the model geometry.
bvhModelOutput unique_ptr containing the loaded model.
outNodeCountOutput number of BVH nodes.
outTriCountOutput number of triangles.
handleOutput BVHHandle whose textures/buffers will be filled.
Returns
True on success, false if the model failed to load.

GLuint & outNodeTex,
GLuint & outNodeBuf,

GLuint & outTriBuf )

Uploads BVH nodes and triangles to GPU texture buffers (TBOs).

Two pairs of objects are created:

  • outNodeBuf + outNodeTex
  • outTriBuf + outTriTex

This design allows shutting down cleanly by deleting buffers while the TBO textures reference them indirectly.

Parameters
nodesFlattened BVH node array.
trisTriangle list associated with the BVH.
outNodeTexOutput: texture buffer containing nodes.
outNodeBufOutput: buffer for node data.
outTriTexOutput: texture buffer containing triangles.
outTriBufOutput: buffer for triangle data.
