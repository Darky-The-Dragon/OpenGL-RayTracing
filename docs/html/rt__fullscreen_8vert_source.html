3/*
4 Fullscreen Triangle Vertex Shader
5
6 This shader generates a fullscreen triangle without requiring any vertex
7 buffers or VAOs. Instead, gl_VertexID is used to procedurally define
8 three clip-space positions that cover the entire screen:
9
10 ( -1, -1 )
11 ( 3, -1 )
12 ( -1, 3 )
13
14 These three points form a single triangle that fully contains the
15 canonical viewport rectangle [-1, 1]². This is a common optimization
16 in modern rendering pipelines because:
17
18 - It avoids sending a fullscreen quad (two triangles) through the pipeline.
19 - It eliminates redundant vertex data, VAOs, and buffer state.
20 - UV computation becomes trivial using the generated clip-space coordinates.
21
22 The shader outputs:
23 - vUV: UV coordinates in [0,1]² derived from clip-space position.
24 - gl_Position: final clip-space position of the fullscreen triangle.
25*/

27out vec2 vUV;

32 // Procedural vertex positions based on gl_VertexID

39 else {
40 p = vec2(-1.0, 3.0);

42
43 // Convert clip-space coordinates ([-1,1]) to UV ([0,1])
